Zimbra is urging customers to patch a critical stored cross-site scripting
(XSS) flaw in the Classic Web Client. Specially crafted emails can execute
malicious scripts within a user's session, giving an attacker a path to
arbitrary actions in the context of the victim's mailbox. The flaw has not
yet been assigned a CVE identifier, and no public reporting of active
exploitation has surfaced. Zimbra Classic Web Client users should apply the
vendor's update once available.
