Google's Threat Intelligence Group (GTIG) identified a cybercrime actor using a zero-day exploit
assessed to have been generated by an AI system — the first confirmed case of AI-assisted exploit
development observed in the wild. The target was an unnamed open-source web administration tool;
the exploit was designed to bypass two-factor authentication and was intended for a mass exploitation
event that Google disrupted before it launched.

The vulnerability was a zero-day at time of discovery, meaning no patch existed when attackers had
a working exploit ready. This marks a meaningful threshold: AI was used not just for reconnaissance
or automation but for actual vulnerability discovery and functional exploit generation. Defenders
should expect the time-to-exploit window for newly disclosed bugs to compress further as AI-assisted
tooling proliferates in the offensive ecosystem.
