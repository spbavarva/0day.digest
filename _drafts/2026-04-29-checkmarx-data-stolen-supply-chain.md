Checkmarx confirmed that attackers exfiltrated data from its GitHub environment on March 30, approximately one week after the threat actors published malicious code as part of the broader supply chain attack. The disclosure moves from suspected breach to confirmed data theft.

The incident is significant because Checkmarx is itself a code security platform used by enterprises for static application security testing. Organizations relying on Checkmarx integrations — particularly GitHub-connected SAST pipelines — should review their integration permissions and monitor for anomalous access patterns. Full details of what data was stolen have not been publicly disclosed.
