A researcher published working proof-of-concept exploit code for multiple Microsoft vulnerabilities to GitHub, making them immediately available to both attackers and security professionals. Microsoft responded publicly by calling uncoordinated zero-day disclosure "never justifiable."

The researcher is reportedly threatening to release additional PoCs. The public availability of working exploit code on a Microsoft-owned platform (GitHub) adds irony and operational urgency — defenders must assume exploitation attempts are imminent for any affected products.

Organizations should prioritize patching any Microsoft vulnerabilities for which PoC code has now been published. The incident also reignites debate about responsible disclosure timelines and the limits of vendor patch cadences.
