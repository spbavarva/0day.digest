Microsoft is adding a two-hour delay before Visual Studio Code automatically installs new versions of installed extensions, as a defense against software supply chain attacks. New extension versions will only auto-update two hours after publication, giving the security community a window to flag malicious releases before they reach most users. The change targets a known weak point in IDE ecosystems, where compromised or malicious extension updates have previously been used to push malware to developers.
